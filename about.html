Who Am I?

Jude the Software Engineer

Who I Was

My path to software engineering began with my studies in music composition. After studying for a couple of years, I decided I wanted to get to work with music. I decided to take a job with MakeMusic, Inc. supporting digital notation software.

Everything from basic To-Do list apps, to the cutting edge of augmented reality interested me - I could not get enough of it. My job was to help people use music software and I loved it; however, I still felt I wanted more. I wanted to build the applications that our customers enjoyed, and I decided to make the switch.

Who I Am

I am an empathetic developer, not only for software users, but also for my team members. I rely on process and its perpetual improvement to produce complex software at pace. It is also important to me that I am able to pair with my teammates when building solutions. I often find that code that Iâ€™ve paired on is some of the best written.

While I enjoy pairing styles like Driver-Navigator, my favorite type of pairing is having one engineer write a test and then having another engineer implement a feature to pass that test.

I have a genuine passion for building well tested software - testing is how I design my code, and I continue to benefit from applying this methodology. My time at the Turing School of Software & Design exercised my abilities to create excellent APIs using both Ruby/Rails and JavaScript/Express using skills like pair programming and Test-Driven Development.

My Tech

My first love was JavaScript. At the time, I was self-teaching web development with HTML, CSS, and JavaScript so I could complete small web page updates and apps for my previous job. Whjen I decided to make the quantum leap and shift my career to software engineering, I decided that building backend technology to serve data to a frontend interface was my niche.

So, I decided to attend the Turing School's backend engineering program which primarily utilized Ruby and Rails. Today, I would mark Ruby/Rails as my primary language/framework. A very close second would be JavaScript, utilizing frameworks like ExpressJS for backend technology and React for frontend interfaces.

I have also completed a project that utilized Elixir/Phoenix to build an API for a provided frontend interface (see Quantified Self for more details). Diving into learning functional programming through use of Elixir made this language and its acommpanying framework (Phoenix) a key part of my coding toolbelt.

I am always ecited to learn new technologies should the need arise. I find that utilizing the right technology for a project is crucial, so I like to keep my mind open about new tech/older tech/any tech! I consider myself a student at all times - if I find myself thinking I know all there is to know about something, then to me it means that I should be asking more questions. There is always more to learn.
